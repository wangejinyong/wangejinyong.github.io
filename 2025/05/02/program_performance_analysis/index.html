2025年05月02日 1.8k 字 大概 8 分钟

程序的性能分析离不开两个维度，一个是时间复杂度，一个是空间复杂度。

时间复杂度：描述了程序的运行时间。

空间复杂度：描述了程序在运行时占用的内存 。

算法导论这样解释：

O用来表示上界，当用它作为算法在最坏的情况下运行时间的上界时，就是对任意数据输入的运行时间的上界。

但是一般来说，人们说一个算法的时间复杂度，是一般情况，而非严格意义上的上界（最复杂的情况）。

例如快速排序，其一般情况与最好情况都是O(nlogn)，但是最坏情况则为O(n^2)。数据用例不一样，算法的时间复杂度也是不尽相同的。

不同数据规模差异

大O表示法并不具体表示代码真正的执行时间，而是表示代码执行时间随数据规模增长的变化趋势。所以也叫渐进时间复杂度。
当n很大时，公式中的低阶、常量、系数三部分并不改变增长的趋势，所以只需要记录最大的量级就好了。

  1. 只需关注循环执行次数最多的一段代码；
  2. 总复杂度等于量级最大的那段代码的复杂度；
  3. 嵌套代码的复杂度等于嵌套内外代码复杂度的乘积。

大O表示法关注增长趋势的量级而非绝对数值差异，最终统一简化为 O(log⁡n)，成为计算机科学中最优雅的抽象之一。

大O表示法到底如何分析一个程序的复杂度？

三类主要的：
1.循环

  • 1层循环 → 通常是 O(n)

    总操作次数 = n → O(n)

  • 2层嵌套循环 → 通常是 O(n²)

    总操作次数 = n × n → O(n^2)

2.递归

  • 每次递归分成两半 → O(log n)（比如二分查找）

    e.g:

    为什么是O(log n)？

    每次循环都将搜索范围 除以2：
    n → n/2 → n/4 → … → 1
    相当于：2^k = n

  • 每次递归分成两份 → O(2ⁿ)（比如斐波那契数列递归）

3.忽略无效项

  • 去掉常数：O(2n + 3) → 直接写 O(n)
  • 只留最大项：O(n² + n + 100) → 只要 O(n²)

常见例子：

e.g. 时间复杂度 why?
i=1 O(1) 赋值
for loop O(n) 一层循环
二分查找 O(log n) 每次去掉一半数据
冒泡排序 O(n^2) 两层循环遍历所有元素两两比较
递归算斐波那契数列 O(2^n) 每次递归分裂成两个子问题

复杂度分析就是：

  1. 找到代码里最费时间的部分（通常是循环或递归）
  2. 数数它和数据量 n 的关系
  3. 忽略小数字，只留最高阶

如图，越往右越慢。

注：在某些情况下，并不是越往左的算法越合适的，具体情况具体分析。

空间复杂度

顾名思义，是程序在运行时占用的内存大小。

和时间复杂度类似，也用大O表示法（如 O(1)、O(n)、O(n²)）来描述。

  • 算法运行过程中 额外申请的变量、数组、递归栈等占用的空间
  • 不包括输入数据本身占用的空间（例如输入数组 nums[] 不算）

常见空间复杂度等级

1. O(1)

算法运行所需的额外空间 固定，不随输入规模 n 变化。
e.g：

  • 只用了一个变量 total：空间复杂度 O(1)

2. O(n)

额外空间占用 和输入规模 n 成正比。
e.g：

  • 分配了一个长度为 n 的新数组：空间复杂度 O(n)

3. O(n²)

常见于二维数组或嵌套数据结构。
e.g：

  • 分配了 n×n 的二维数组——空间复杂度 O(n^2)

4. O(log n)

通常出现在 递归算法 中，递归深度为 log n。

  • 递归深度是 log₂n（每次数据减半）：空间复杂度 O(logn)

经典算法的空间复杂度分析

e.g. 时间复杂度 空间复杂度 why?
二分查找 O(log n) O(1) 只用几个变量
归并排序 O(n log n) O(n) 需要临时数组合并
快速排序 O(n log n) O(log n) 递归栈深度
斐波那契递归 O(2^n) O(n) 递归树深度最多n层

结语：

最好的代码不一定是最高效的，但一定是复杂度可控的。

下次遇到算法问题，不妨先问自己：

“我的代码时间够快吗？”（时间复杂度）

“我的代码内存够省吗？”（空间复杂度）
